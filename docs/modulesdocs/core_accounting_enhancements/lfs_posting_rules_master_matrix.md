# LFS Posting Rules Master Matrix

## Event → Debit → Credit → Dimensions

System: LFS -- Logistics Financial System\
Version: 1.0\
Purpose: Enterprise GL Posting Rules Reference for Core Accounting

------------------------------------------------------------------------

# 1. Introduction

This document defines the **Posting Rules Master Matrix** used by the
LFS Core Accounting Engine.

Each financial event generated by operational modules is mapped to a
**standard accounting rule** consisting of:

Event → Debit Account → Credit Account → Dimensions

The Posting Rules Engine reads these rules and automatically generates
journal entries.

------------------------------------------------------------------------

# 2. Dimension Tags

Journal lines support the following profitability dimensions:

client_id\
shipment_id\
route_id\
warehouse_id\
vehicle_id\
project_id\
service_line_id\
cost_center_id

These dimensions allow financial reporting such as:

-   Shipment profitability
-   Client profitability
-   Route profitability
-   Warehouse profitability

# 15. Usage in Posting Rules Engine

Workflow:

Financial Event\
→ Rule Lookup\
→ Debit/Credit Accounts\
→ Dimension Resolution\
→ Journal Builder\
→ JournalService Posting

# 17. Governance

Posting rules must be managed by finance administrators.

Required controls:

-   rule versioning
-   approval workflow
-   audit trail
-   activation/deactivation
